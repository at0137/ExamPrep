11.04 | Tackling Big Data with Data Science

モジュール11：ビッグデータ技術入門（Introduction to Big Data Techniques）11.04｜データサイエンスによるビッグデータへの対応（Tackling Big Data with Data Science）

 Learning Outcome

The candidate should be able to:

  • 投資運用（investment management）におけるビッグデータ（Big Data）およびデータサイエンス（Data Science）の応用を説明する。

データサイエンス（data science）は、コンピュータサイエンス（computer science）（MLを含む）、統計学（statistics）、その他の分野の進歩を活用して、ビッグデータ（Big Data）（または一般のデータ）から情報を抽出することを目的とした学際的分野として定義される。企業は、さまざまなビジネスおよび投資目的のために、データサイエンティストやアナリストの専門知識に依存してビッグデータから情報や洞察を抽出している。

データサイエンティストにとって重要な考慮事項の1つは、データの構造である。ビッグデータの議論で述べたように、非構造的な性質を持つため、代替データ（alternative data）は分析に使用する前に特別な処理を必要とする場合が多い。

データ処理手法（Data Processing Methods）

ビッグデータ分析に適したデータ管理手法を決定するために、データサイエンティストは、キャプチャ（capture）、キュレーション（curation）、保存（storage）、検索（search）、転送（transfer）などのさまざまなデータ処理手法を用いる。

  • キャプチャ（capture）—データキャプチャとは、データがどのように収集され、分析プロセスで利用可能な形式に変換されるかを指す。低遅延システム（low-latency systems）—最小限の遅延で大量のデータを通信するネットワーク上で動作するシステム—は、リアルタイムの価格や市場イベントに基づいて意思決定を行う自動取引（automated trading）に不可欠である。一方で、高遅延システム（high-latency systems）はリアルタイムデータへのアクセスを必要としない。
  • キュレーション（curation）—データキュレーションとは、データクレンジング（data cleaning）を通じてデータの品質と正確性を確保するプロセスを指す。このプロセスでは、すべてのデータを確認して誤り（不正確または不適切なデータ）を検出・特定し、必要に応じて欠損データを補正する。
  • 保存（storage）—データ保存とは、データがどのように記録・アーカイブ・アクセスされるか、および基盤となるデータベース設計を指す。重要な考慮事項は、データが構造化（structured）か非構造化（unstructured）か、また分析ニーズに低遅延ソリューションが必要かどうかである。
  • 検索（search）—検索とは、データに対してどのようにクエリ（query）を実行するかを指す。ビッグデータは、大量のデータを調査・確認して必要な情報を特定するための高度なアプリケーションの必要性を生み出している。
  • 転送（transfer）—転送とは、データが基盤となるデータ源または保存場所から分析ツールへどのように移動するかを指す。これは、証券取引所の価格フィードのような直接的なデータフィードを通じて行われる場合がある。

データ可視化（Data Visualization）

データ可視化（data visualization）はビッグデータ理解のための重要なツールである。可視化とは、データをどのようにフォーマットし、表示し、グラフィカルに要約するかを指す。従来の構造化データは表やチャート、トレンドで可視化できるが、非構造的データには新しい可視化手法が必要である。例えば、インタラクティブな3次元（3D）グラフィックスでは、特定のデータ範囲に焦点を当てたり、3D軸でデータを回転させたりして、トレンドや関係性を特定できる。3変数以上の多次元データ分析には、色、形状、サイズなどを追加するなどの手法が必要となる。また、ヒートマップ（heat maps）、ツリーダイアグラム（tree diagrams）、ネットワークグラフ（network graphs）など、データ構造を視覚的に表現する多様な手法が存在する。

テキストデータに適用される可視化手法として「タグクラウド（tag cloud）」がある。これは、単語の出現頻度に応じて文字サイズを変えて表示するものである。頻度の高い単語は大きく、低い単語は小さく表示される。また、「マインドマップ（mind map）」も別の手法であり、単語の頻度ではなく概念間の関係性を示す。

フィンテック（fintech）は投資運用の多くの分野で活用されており、テキスト分析（text analytics）や自然言語処理（natural language processing）、リスク分析（risk analysis）、アルゴリズム取引（algorithmic trading）などに応用されている。

テキスト分析と自然言語処理（Text Analytics and Natural Language Processing）

テキスト分析（text analytics）は、企業提出書類、レポート、決算説明会、ソーシャルメディア、電子メール、インターネット投稿、調査などの大規模で非構造的なテキスト・音声データから意味を抽出するためにコンピュータプログラムを用いる手法である。また、異なるデータ源から自動的に情報を取得して意思決定を支援する。より高度な分析としては、語彙分析（lexical analysis）やキーワード・フレーズに基づくパターン認識が含まれる。将来のパフォーマンス指標（例：消費者センチメント）の特定にも利用される。

自然言語処理（natural language processing, NLP）は、コンピュータサイエンス、AI、言語学の交差領域であり、人間の言語を分析・解釈するプログラムの開発を目的とする。翻訳、音声認識、テキストマイニング、センチメント分析、トピック分析などが含まれる。また、コンプライアンス用途として、従業員の通信内容を監視し、不正行為や規制違反の検出、機密情報の保護にも利用される。

世界中の公開企業は毎年膨大な報告書と決算説明会を生成しており、個人やチームでは処理しきれない量である。NLPはMLと組み合わせることで、これらのデータを短時間かつ高精度で分析し、トレンドを特定できる。

例えば、アナリストのコメント分析にNLPを用いることで、投資判断を支援できる。EPS予測だけでは見落とされるニュアンスを分析し、センチメント（sentiment）評価を付与することで、推奨変更前の兆候を捉えることができる。

同様に、欧州中央銀行（European Central Bank）や米連邦準備制度（US Federal Reserve）などの政策当局の発言分析にも応用される。NLPは語調や語彙の微妙な違いを分析し、金利政策やインフレ期待などのトレンドを把握する。

NLPモデルは、人々の意見や嗜好を分析し、企業や市場の短期トレンド予測に活用される。例えば、TwitterのセンチメントとIPOパフォーマンスの関係などが研究されている。

  Programming Languages and Databases

データサイエンスで使用される主なプログラミング言語には以下がある。

  • Python：オープンソースで無料の言語であり、高度な分析アプリケーションの開発に広く利用されている。
  • R：統計分析向けのオープンソース言語であり、MLや計量経済学などに対応するパッケージを持つ。
  • Java：多様な環境で動作するプログラミング言語であり、多くのインターネットアプリケーションで使用される。
  • CおよびC++：高速処理に適したプログラミング言語であり、アルゴリズム取引や高頻度取引に用いられる。
  • Excel VBA：マクロによる自動化を通じてデータ処理とプログラミングを橋渡しするツールであり、カスタム分析やレポート作成に利用される。

使用される代表的なデータベースの種類には以下がある。

  • SQL：SQLは、行と列を持つテーブルにデータを格納できる構造化データ（structured data）のためのデータベースクエリ言語（database query language）である。SQLベースのデータベースはサーバー上で動作し、ユーザーはSQLクエリを用いてアクセスする。
  • SQLite：SQLiteは構造化データ（structured data）のためのデータベースである。SQLiteデータベースはプログラム内に組み込まれ、サーバー上で動作させる必要がない。データアクセスを必要とするモバイルアプリで最も一般的に使用されるデータベースである。
  • NoSQL：NoSQLは、行と列による従来のテーブル形式では表現できない非構造化データ（unstructured data）のためのデータベースである。

コンピュータサイエンス（computer science）、統計学（statistics）、その他の分野の進歩を活用して、ビッグデータ（Big Data）（または一般のデータ）から情報を抽出することを目的とする学際的分野。

企業提出書類、レポート、四半期決算説明会、ソーシャルメディア、電子メール、インターネット投稿、調査などの大規模で非構造的なテキストまたは音声データセットから意味を分析・抽出するためにコンピュータプログラムを使用すること。

テキスト分析（text analytics）の分野に属し、コンピュータサイエンス、人工知能（AI）、言語学の交差領域に位置する研究分野であり、人間の言語を分析・解釈するコンピュータプログラムの開発に焦点を当てる。
